### 3.1 Agent Invocation Protocol（已存 memory）

→ 母敘事化版本：**修真者帶靈寵進福地探索**。
→ 技術原型已存在（[[project_agent_invocation_protocol]]）、現在有了世界觀外殼。

### 3.2 SS 原有定位（PROJECT.md）
> SS 是 LL 生態的競技大廳、玩家上傳 RR 訓練好的 agent 升格為靈將、進入各境對戰。

→ 母敘事擴充：
- 原本「各境」是抽象的「遊戲列表」、現在每個境都是**有完整世界觀的福地洞天**
- 「上傳靈將」不是孤立動作、是「**靈氣復甦下、修真者帶靈寵入世**」的具象
- SS 不再只是 UI 殼、是**整個 LL 宇宙觀的敘事入口**

### 3.3 三 IP 統一宇宙（[unified-universe.md](ip/unified-universe.md)）
> 造物森林 + GlowFrame + HH 共享核災起源 + 雙盲反轉。

→ 在母敘事框架下：
- 這三個 IP 構成 SS 中的**第一個福地群**（互相串連的小世界）
- HH 是這個福地群的中心
- 未來其他 IP 可以是**獨立的福地**、不必跟這個群串連

### 3.4 三軸架構（LL / II / AA）
- **LL 軸**：成就認證、修真者的修為等級
- **II 軸**：陪玩 agent 群星、修真者的靈寵團
- **AA 軸**：avatar 基因序列、靈寵的 DNA 傳承
- **SS** = 三軸交會處——修真者（LL）帶靈寵（II + AA）進福地（SS）

## 四、為什麼這層母敘事重要

### 4.1 對使用者
- 不需要先理解技術、聽「靈氣復甦」就懂
- 帶 agent 不是「呼叫 API」、是「**召喚靈寵**」——情感連結強得多
- 進 IP 不是「玩遊戲」、是「**入福地修練**」——有修真感、有成長感

### 4.2 對行銷
- 一句話 logline：「**AI 時代的修真世界**」
- 不需要解釋 RL / agent / orchestration、修真小說讀者全懂
- 中文圈廣大的仙俠 IP 用戶基礎、可直接共鳴

### 4.3 對 IP 擴充
- 每個新 IP 只要寫**「這是哪個福地、有什麼修真法則」**就夠
- 不需要跟其他 IP 強行串連
- 三 IP 統一宇宙是「**福地群**」、其他 IP 可以是「**獨立洞天**」

### 4.4 對 LL 整體 vision
- LL 不再是「公司」、是**「靈氣復甦時代的修真宗門」**
- LL 用戶不再是「客戶」、是「**門下弟子**」
- LL agent 不再是「工具」、是「**師徒共修的靈寵**」
- → 這就是 [[project_membership_paradigm_shift]] 講的「**被記得而非能用**」的世界觀載體
