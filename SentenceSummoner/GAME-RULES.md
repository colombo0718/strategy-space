## ★ 設計第一原則（設計羅盤）

> **遊戲裡每一個有意義的行動，都由「產出一個單字／一句話」來驅動；而且靠鼓勵、不靠強迫。**

- **鼓勵、非強迫**：低地板——唸出/打出卡上的字就有回饋、就有進展；高天花板——說得越通順/越有創意、獎勵越多。讓小孩**想**開口，不是**被逼**開口。
- **語言學習內建、跑不掉**：只要在玩，就一定在唸字、拼句、練發音——教育效果是規則的副產物，不是外加的說教。
- **檢驗每個機制**：任何能讓玩家「不開口就推進」的設計 → 稀釋核心價值 → 改掉或砍掉。
- **動機層 vs 核心**：怪獸／血量／棋盤／星點都是「動機層」，存在只為讓人**想再開口一次**。→ 先立核心迴圈（產出語言 → 回饋），再疊動機層。

## 0. 一句話

雙方用**相同的 64 張英文牌組**，輪流**打句子**驅動所有行動——召喚怪獸上場、指揮怪獸攻擊、用形容詞強化/弱化——把對方**召喚師打到 HP 歸 0** 獲勝。
**打出/唸出卡上的字**就能行動（練發音/拼字）；日後 **LLM 裁判**再對「通順、有創意」的句子額外加成（練造句）。

## 3. 回合流程（輪流，一方一回合）

**一回合可以做很多動作**——想得到、牌到位，就一直打句子做下去，做完按「**結束回合**」或在輸入框打 `end`。

一回合內可自由組合：
1. **召喚**：把手牌名詞召上場（一句可多重召喚，見 §5）。新召喚的怪獸**當回合睡眠、不能行動**。
2. **攻擊**：指揮**上回合就在場（非睡眠）**的怪獸攻擊（基礎攻擊免卡、或用動詞卡加成，見 §5、§6）。每隻怪獸一回合出手一次。
3. **形容詞**：正向強化自己怪獸、負向弱化對方怪獸（永久常駐，見 §5）。
4. **防禦動詞卡**：強化自己怪獸的 🛡️/🦶/❤️。
5. **結束回合**：按鈕或打 `end`／`pass`／`結束`（此指令也是給 AI/LLM 用的）。

**出牌補抽**：每打出一張牌（召喚/動詞/形容詞）就**自動補抽 1 張，手牌維持 5**（牌庫還有牌就補）。
→ 保證「不會沒牌卡住玩不下去」。基礎攻擊不耗卡、不補抽。

> 沒有「一回合只能一次攻擊」的限制。名詞召喚後成為場上怪獸、不再回到手牌。

## 5. 造句機制（句子驅動、關鍵字裁判）

**裁判＝程式關鍵字比對**：句子裡出現手牌/場上的**單字**就發動對應行動（目前不判文法通順，只認單字有沒有到）。

### 5.1 召喚（名詞）
- 句中出現手牌**名詞** → 召喚上場。例：`Cat` / `I summon Cat`。
- **多重召喚**：一句點多個名詞可一次全召。例：`Cat and Dog and Fox`（超過場上上限的留在手牌並提示）。
- 新召喚的怪獸**當回合睡眠**（不能攻擊/行動），下一回合起才行動。

### 5.2 攻擊（怪獸 + 動作 + 敵方）
主詞 S＝**你場上非睡眠的怪獸**；受詞 O＝**對方場上的怪獸**（或其召喚師）。
- **基礎攻擊（免動詞卡）**：句中有一般攻擊字（`attack/hit/fight/strike/punch/smash/charge`…）即可。例：`Cat attacks Dragon`。傷害＝怪獸自身 ATK。
- **動詞卡加成**：句中打出**攻擊動詞卡**（Bite/Bump/Kick…，帶 🗡️）→ 傷害額外加該卡 🗡️。例：`Cat bites Dragon`。動詞卡用掉、補抽。
- **擋牆規則**：對方場上有怪獸時，**必須先指定打一隻對方怪獸**；把對方怪獸清空後，攻擊才能**直擊召喚師**。
- 怪獸每回合出手一次（出手後該回合不再行動）。

### 5.3 形容詞（正/負向、一刀切、永久）
- **用數值判正負**：效果全為正 → **正向**；含任一負值 → **負向**。
- **正向只強化自己**：`Strong Tiger`（順手也可召喚該名詞再強化）。
- **負向只弱化對方**：`Panda sick` → 弱化對方場上的 Panda（不用講 "Your"，負向自動找對方；打不到自己＝無誤傷）。
- **永久常駐**：形容詞卡疊在該怪獸上、持續生效；四維修正直接改怪獸的 ATK/DEF/HP/SPD。怪獸陣亡時，疊著的形容詞一起進棄牌。
- 目前 16 張形容詞**恰好 8 正 8 負、無混合卡**。若日後出現同時有正有負的混合卡，再回頭談分層規則。
- **❤️ 負向形容詞＝詛咒**（僅 **Weak ❤️-2**、**Hurt ❤️-1** 兩張）：**同時砍「總生命上限」與「當前生命」**，❤️ 歸零即**死亡**（跟被打死一樣清場、身上的卡進棄牌）。因為砍了總血上限，**之後就算用治療動詞也補不回原本的血量**——是真正的削弱。

### 5.4 防禦/輔助動詞卡（**暫時** buff，與永久形容詞區分）
- 動詞卡若**無 🗡️、只有 🛡️/🦶/❤️**（Guard/Dodge/Curl/Protect/Run/Block/Heal/Fly）→ 對**自己怪獸**生效。例：`Penguin blocks`。
- **🛡️（Curl/Guard/Block）＝暫時護甲**：當場加到護甲、擋當前這波攻擊；**只持續到你下一回合、之後解除**（時序：用的當回合＝滿甲、下一輪仍在、再下一輪恢復原值）。
- **🦶（Dodge/Run/Fly）＝暫時加速**：提高該怪獸閃避、同樣一回合後解除。
- **❤️（Heal/Protect）＝即時治療**：補當前 HP、不超過上限（不是暫時的）。
- **對比**：動詞是「一次性防禦姿態」（暫時）；**形容詞才是永久 build**（§5.3）。

### 5.5 結束回合
- 按「結束回合」鈕，或在輸入框打 `end`／`pass`／`結束`。此文字指令讓 AI/LLM 也能用同一介面結束回合。

## 6. 結算公式（*可調*）

一次攻擊分三步：

**A · 命中判定（速度閃避）**
```
速度差 d = SPD_守 − SPD_攻
d ≤ 0（攻方不慢）→ 必中
d ≥ 1（守方較快）→ 守方擲 1d6，擲值 ≤ min(d, 5) → 閃過（0 傷害）
```
（d ≥ 5 時最高 5/6 機率閃，仍保留至少 1/6 命中機會。）

**B · 攻擊力（純加、無下限修飾——讓學生看得懂怎麼算）**
```
攻擊力 = ATK_攻 + 動詞🗡️加值
```
- 基礎攻擊（免卡）：動詞加值 = 0 → 攻擊力 = ATK。
- 攻擊動詞卡：加該卡 🗡️。
- **刻意不加 max(1)**：傷害要是透明的加減法，不放隱藏下限。

**C · 護甲吸收與傷害（護甲＝每回合的護盾池）**
```
護甲吸收 = min(當前護甲, 攻擊力)      ← 吸收多少就消耗多少護甲
打怪獸：傷害 = 攻擊力 − 護甲吸收       ← 吸收 ≤ 攻擊力，傷害自然 ≥ 0，不需 max
直擊召喚師：召喚師 HP − 攻擊力         ← 召喚師無護甲、吃滿
```
- 每隻怪獸的「護甲」＝其 DEF，於**其控制者回合開始時回滿**。
- 攻擊會**消耗**目標護甲；**同一回合連續攻擊同一目標 → 護甲耗盡後直接打穿（集火破甲）**；下回合回滿。
- 弱攻擊被完全吸收 = 0 傷害，但**護甲照樣被消耗**（所以連戳也能磨開）。
- 護甲被消耗時，怪獸卡上顯示剩餘 🛡️ 值；戰報顯示 `🗡️5−🛡️3 ＝ −2　💢護甲破！`。
- 怪獸 current HP ≤ 0 → 陣亡進棄牌。
- **溢出傷害（踐踏）**：攻擊打死怪獸時，**超出目標剩餘 HP 的傷害會溢到對方召喚師**（衝破防線）。例：Duck 剩 1 血、被打 4 → Duck 死、多的 3 直擊召喚師 −3。這是打穿滿場防線、逼出勝負的主要壓力來源。

> **設計理由**：舊制「DEF 每一擊都固定減傷」→ 高甲肉盾每擊都擋、怎麼打都磨。改成護盾池後，DEF 變成「一回合擋一波」的資源，**集火可破、單戳磨不動**，給多怪獸站場的集火戰術一個回報；且傷害是純減法、無隱藏下限，教學透明。

**D · 裁判倍率（規劃 · 天花板）**
> 尚未接。日後由 **LLM 裁判**對整句評級，給傷害倍率——把「找字」升級成「造句」的獎勵：

| 裁判評級 | 條件 | 倍率 |
|---|---|---|
| Tier 0 | 只有字／不成句 | ×1.0 |
| Tier 1 | 通順、語意合理 | ×1.3 |
| Tier 2 | 通順且有創意/生動 | ×1.6（暴擊感） |

```
（規劃）最終傷害 = round( 步驟C 傷害 × 裁判倍率 )
```
- 裁判只給**加成**、不擋行動（呼應「鼓勵不逼」）。基礎命中仍由程式關鍵字判。

## 7. 裁判與對手

### 現況：純腳本（已實作）
- **裁判**：程式關鍵字比對——句中出現手牌/場上單字就發動對應行動（§5）。涵蓋目前**全部規則**，不需 LLM。
- **對手**：腳本 AI（`aiTurn`）——**一回合做多個動作**：召喚補到 3 隻 → 正向形容詞疊自己攻擊最高的、負向疊你場上攻擊最高的 → 每隻可行動怪獸都攻擊（會挑「能一擊打死」或防禦最低的目標，清場則直擊召喚師）。強度旋鈕 `AI_BOARD`。

### 規劃：LLM 雙角色（Cloudflare Workers AI）
兩個獨立、互不干涉的位置，可分開接：
- **① 裁判**：評玩家句子「通順/有創意」給 Tier 倍率（§6-D）。寬鬆友善、只加成不擋。
- **② 對手**：讀盤面自己造 SVO 句出招，比腳本更活。用同一套 §5/§6 結算；也用 `end` 指令結束回合。
- **STT（語音）**：`@cf/openai/whisper-large-v3-turbo` 轉文字 → 程式關鍵字判 → 裁判評 tier。可加「聽到的是 ___，對嗎？」讓小孩確認、順便校正發音。

## 9. 遊戲三模式（教學 → 訓練 → 對戰）

低地板 → 高天花板的掌握階梯，全部由「產出句子」驅動（見 §★）：

| 模式 | 目標 | 資訊公開 | 對手 | 定位 |
|---|---|---|---|---|
| **教學模式** | 教會遊戲規則 | 全公開＋導引 | 腳本情境 | 最低地板：擺好情境、一步步帶著寫句子召喚/攻防 |
| **訓練模式** | 實戰練習 | **AI 手牌攤開** | 腳本 AI（可讀）→ 日後 LLM | 中段：真對戰但看得到 AI 的牌＋句子，AI 當語言身教 |
| **對戰模式** | 競技 | **蓋牌** | AI(藏手) / 真人 PvP | 最高天花板：隱藏資訊、鬥智 |

**實作關係**：三模式共用同一套引擎（`en/play.html`），用網址參數 `?mode=teaching/training/battle` 切換。
- **訓練 ↔ 對戰** 只差一個旗標：對手手牌 **攤開 / 蓋牌**（`AI_OPEN`）。
- **教學** ＝ 在引擎上加一層**腳本情境**（預設盤面＋手牌＋目標句、逐步導引）——唯一需「作者撰寫內容」的一塊，工程量最大、對少兒最關鍵。**尚未撰寫。**
