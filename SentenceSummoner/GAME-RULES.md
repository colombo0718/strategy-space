# Sentence Summoner — 遊戲規則書 v1（可照此實作 Demo）

> 言出法隨 / Sentence Summoner。搭配 [`DECK-DESIGN.md`](DECK-DESIGN.md)（卡牌數值規格）。
> 本檔定義**完整遊戲順序**，供 kids-ai-lab「人 vs AI」語音造句 Demo 直接實作。
> 標「*v1 預設（可調）*」的數字是平衡旋鈕，先用這組跑起來、再依實測調。

---

## ★ 設計第一原則（設計羅盤）

> **遊戲裡每一個有意義的行動，都由「產出一個單字／一句話」來驅動；而且靠鼓勵、不靠強迫。**

- **鼓勵、非強迫**：低地板——唸出卡上的字就有回饋、就有進展；高天花板——說得越通順/越有創意、獎勵越多。讓小孩**想**開口，不是**被逼**開口。
- **語言學習內建、跑不掉**：只要在玩，就一定在唸字、拼句、練發音——教育效果是規則的副產物，不是外加的說教。
- **檢驗每個機制**：任何能讓玩家「不開口就推進」的設計 → 稀釋核心價值 → 改掉或砍掉。
- **動機層 vs 核心**：怪獸／血量／棋盤／星點都是「動機層」，存在只為讓人**想再開口一次**、不是價值本身。→ 先立核心迴圈（產出語言 → 天裁回饋），再疊動機層。

## 0. 一句話

雙方用**相同的 64 張英文牌組**，輪流用手牌組出 **SVO 句子**攻擊對方角色。
**唸出卡上的字**就能出招（練發音）；**串成有意義的句子**額外加成（練造句）。
對手與裁判都由 **Cloudflare Workers AI 的 LLM 一人分飾兩角**。

---

## 1. 對局準備

- **牌組**：每方各持一份**相同的 64 張牌**（32 名詞／16 動詞／16 形容詞），**各自洗牌、各自抽**（對稱，不共牌庫）。
- **起手**：各抽 **5 張**手牌。*v1 預設（可調）*
- **上場角色**：開局各從手牌**召喚 1 個名詞**當「上場角色」（見 §4 星點分配）。雙方各 1 個角色對戰。
- **隊伍/命**：每方以 **3 個名詞為一隊**（＝3 條命）；角色陣亡就從手牌召下一個，**3 個全滅則落敗**。*v1 預設（可調）*

---

## 2. 場上結構

```
┌─ 對手(AI) ─ 上場名詞[HP/ATK/DEF/SPD] · 手牌(背面) · 剩餘命 ●●○
│
├─ 中央 ─ 出招造句區 / 骰子 / 天裁講評
│
└─ 玩家 ─ 上場名詞[HP/ATK/DEF/SPD] · 手牌(正面) · 剩餘命 ●●●
```

- 每方**檯面同時只有 1 個上場名詞**（主戰角色）。
- 角色的四維（HP/ATK/DEF/SPD）即時顯示；HP 歸 0 即陣亡。
- 形容詞造成的持續 buff/debuff 掛在角色身上顯示。

---

## 3. 回合流程（輪流，一方一回合）

1. **抽 1 張**（開局第一回合不抽）。手牌上限 **7**，超過於回合結束棄到 7。*v1 預設（可調）*
2. **（可選）換角色**：若無上場角色（剛陣亡）→ 免費召喚新名詞；若主動換人 → **消耗本回合攻擊**。
3. **造句攻擊**（本回合核心，見 §5）：打出 **1 張動詞卡**（＝招式），**可再加最多 1 張形容詞卡**（buff 自己或 debuff 對方）。
4. **語音／文字輸入句子** → **判定命中＋天裁加成**（見 §5、§6）。
5. **結算傷害**（見 §6），更新 HP／持續效果。
6. 打出的動詞卡進棄牌堆；形容詞若為「持續」則留在角色上。**換手**。

> **一回合＝1 次攻擊**（1 動詞 ＋ 最多 1 形容詞）。名詞是檯面角色、不重複打出。

---

## 4. 星級＝開局分配點（讓稀有度有玩法用途）

- 召喚名詞上場時，依星等獲得點數：**1★=1／2★=2／3★=3**。
- 自由加到四維（HP/ATK/DEF/SPD），**公開明配、無單項上限、召喚後鎖定**。
- 同一張名詞因分配不同 → 長出不同 build（額外數學＋變化性）。

---

## 5. 造句攻擊機制

**SVO 對應卡牌**：
- **主詞 S** ＝ 你的**上場名詞**（如 Cat）
- **動詞 V** ＝ 手牌打出的**動詞卡**（如 Bite）← 這是招式本體
- **受詞 O** ＝ 對方的**上場名詞**（如 Dragon）
- **（可選）形容詞** ＝ 手牌形容詞卡，修飾（"**Fast** Cat bites Dragon" / "Cat bites **Scared** Dragon"）

→ 玩家要表達的目標句：**「Cat bites Dragon」**（＋可選形容詞）。

### 語音兩層命中（英文版核心）

| 層 | 要求 | 由誰判 | 效果 |
|---|---|---|---|
| **① 基礎命中（地板）** | 語音中**唸出動詞卡的字**（如 "bite"） | **程式**：Whisper 轉文字後**字串比對**（寬容、模糊比對、破英文也認） | 招式生效，進入結算 |
| **② 造句加成（天花板）** | 唸出的內容**串成有意義的句子** | **LLM 天裁**評整句 | 傷害倍率加成（見 §6） |

- **唯一「gate」**：動詞的字沒被偵測到 → **攻擊落空**（本回合攻擊作廢）。其餘一律不擋，鼓勵開口。
- **文字輸入備援**：吵雜／無麥克風時可打字，走同一套判定（跳過 Whisper、直接進天裁）。
- **難度可調**：小小孩「只唸一個動詞」即可攻擊；大孩子「完整 SVO」拿滿加成。

## 6. 結算公式（*v1 預設，全可調*）

**步驟 A · 命中判定（速度）**
- 速度差 `d = SPD_守 − SPD_攻`
- `d ≤ 0`（攻方不慢）→ **必中**
- `d ≥ 1`（守方較快）→ 守方擲 **d6 閃避**：擲值 ≤ `d` 則**閃過**（未命中）；上限 5（永遠有 1/6 命中）

**步驟 B · 基礎傷害**
```
基礎傷害 = max(1, (ATK_攻 + 動詞🗡️加值) − DEF_守)
```
- 動詞攻擊型提供 🗡️+1~+3（作用一回合）；防禦/輔助型動詞改為加自己 🛡️/🦶/❤️（見 DECK-DESIGN §3）。

**步驟 C · 天裁倍率（造句加成）**
| 天裁評級 | 條件 | 倍率 |
|---|---|---|
| Tier 0 | 只有字／不成句／不合理 | ×1.0 |
| Tier 1 | 通順、語意合理 | ×1.3 |
| Tier 2 | 通順且有創意/生動 | ×1.6（暴擊感） |

```
最終傷害 = round( 基礎傷害 × 天裁倍率 )
```

**步驟 D · 形容詞**（若本回合打出）
- buff：加自己某維 +1~+2（持續，留在角色上）
- debuff：扣對方某維 −1~−2（持續）
- 於步驟 B 之前套用當前所有持續修正。

## 7. AI 對手＝LLM 雙角色（Cloudflare Workers AI）

同一個 LLM 擔任**兩個角色**，兩個獨立 prompt 契約：

### 角色 A · 天裁（判玩家的句子）
```
輸入：{ transcript(玩家語音轉文字或打字), intended:{subject, verb, object, adjective?} }
輸出：{ tier: 0|1|2, comment: "給小孩的一句鼓勵/講評(英文)" }
```
- 只評「通不通順、合不合理、有沒有創意」，**寬鬆友善、鼓勵開口**。
- 基礎命中（動詞字有沒有唸到）由**程式**先判，天裁只決定 tier。

### 角色 B · 對手（AI 自己出招）
```
輸入：{ aiHand:[cards], aiActive:名詞, playerActive:名詞, board狀態 }
輸出：{ verb, adjective?, sentence:"合法 SVO 英文句", targetObject:playerActive }
```
- AI 從手牌挑一張動詞（±形容詞），造一句**攻擊玩家角色**的 SVO 句。
- 難度旋鈕：低階＝挑弱動詞/簡單句；高階＝挑強動詞/生動句（呼應 Boss 模式「越能說會道越難打」）。
- AI 出的句子同樣走 §6 結算（AI 句子預設 Tier 1；高難度可讓它衝 Tier 2）。

### STT（語音）
- `@cf/openai/whisper-large-v3-turbo` 轉文字 → 程式做動詞關鍵字偵測（基礎命中）→ 天裁評 tier。
- 可加「聽到的是：___，對嗎？」讓小孩確認再送出（順便自我校正發音）。

## 8. 勝負

- 對方**上場角色 HP 歸 0** → 陣亡，對方召下一個名詞。
- 一方**3 個名詞全滅**（且無法再召）→ **落敗**。*v1 預設（可調）*

---

## 9. 難度／年齡調節（掛在同一套規則上）

| 級別 | 出招要求 | 天裁 | 對手 |
|---|---|---|---|
| 幼幼 | 唸出 **1 個動詞** | 幾乎全給 Tier 1 鼓勵 | AI 出弱招、簡單句 |
| 進階 | 完整 **SVO 句** | 嚴一點、獎勵創意 | AI 出強招、生動句 |
| Boss/PvE（後續） | 多人合作對高血 Boss | Boss 更會說會道更難打 | 對應 SS「破境試煉／鎮塔將靈」 |

## 10. MVP Demo 範圍（第一版先做這些）

**做**：
- 單頁 `play.html`（英文版），沿用 `en/deck.js` 資料與 `en/deck.html` 卡面樣式。
- 各抽 5 張、召喚 1 上場名詞（先跳過星點分配 or 給簡化版）。
- 玩家**打字或語音**造句 → Whisper 動詞偵測 → LLM 天裁 tier → §6 結算。
- AI 對手用 LLM 出一句攻擊 → 結算。
- 單一上場角色對戰、HP 歸 0 分勝負（先不做 3 命隊伍、不做完整牌庫循環也行）。
- 目標：**跑通「造句→AI 評判/出招→結算」核心循環**。

**先不做（第二階段再疊）**：3 命隊伍、完整抽牌棄牌、星點分配 UI、形容詞持續效果、Boss/PvE、中文套、印刷 PDF。

## 11. 待定／開放問題（等 colombo 拍板）

- 起手 5 張、手牌上限 7、隊伍 3 命——數字先用 v1，實測再調。
- 星點分配要不要進 MVP（或 MVP 先固定四維、第二版再開放分配）。
- 天裁倍率（1.0／1.3／1.6）與速度閃避曲線的手感，需對戰實測。
- 語音「確認轉錄」步驟要不要預設開（教學價值 vs 流暢度）。
- AI 對手難度是否給玩家可選（幼幼／進階）。
