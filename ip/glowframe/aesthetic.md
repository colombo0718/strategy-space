# 流螢戰甲：美術風格

## 一句話

> **黑底 + 單色光條** = 整套 IP 的視覺定錨。

## 起源（從技術限制變成風格）

colombo 原話：

> 「原本是早年計算機算圖能力不足、僅以簡單的光條勾勒出形狀導致。後成為了科幻與賽博龐克的標配。」

→ **技術限制 → 風格特色**——跟 LL 整體哲學一脈：免費 / 受限資源、巧妙運用變優勢。

## 對標美術參考

| 作品 | 借鏡點 |
|---|---|
| **創 光速戰記**（Tron: Legacy）| 黑底 + 光條核心語言 |
| Cyberpunk 2077 | 光條 + 街頭文化 |
| Daft Punk 視覺 | 光條 + 極簡 |
| 武俠電影（如「俠女」、「臥虎藏龍」）| 武術動作、傳統器械 |
| 樂園追放（動漫）| 後人類 + 戰甲 + 武術 |

## 視覺三要素

### 1. 黑底
- 主色 = 純黑（或極暗）
- 環境、戰甲本體都是黑
- 凸顯光條的閃耀感

### 2. 單色光條
- **每副戰甲一個主色光條**——這是區分戰甲身份的方式
- 初號機 = 紫光條
- 未來戰甲 = 紅 / 藍 / 綠 / 金 / 白 / 青 等
- 光條描繪輪廓、關節、武器邊緣

### 3. 細節錯覺
- 「**以為很露但其實都是一坨黑**」（colombo 原話）
- 黑色身體 + 光條輪廓 = 視覺上彷彿能看到細節、實際只有剪影
- 這個設計**省 voxel 工程量**、且觀眾自己腦補細節
- 跟早期 vector graphics 同邏輯

## 武俠妝造 → 流光的拓展邏輯

colombo 原話：
> 「之後要快速拓展這個世界觀也很簡單、就是原本武俠的妝造素材、套成『創』這種科幻流光就好。」

### 具體操作

```
傳統武俠角色設計
  ├─ 服裝（袍 / 鎧 / 衣）
  ├─ 武器（劍 / 刀 / 槍 / 鞭 / 棍 / 暗器）
  ├─ 配件（玉佩 / 髮飾 / 面紗 / 鬥笠）
  └─ 動作（揮劍 / 拔刀 / 點穴 / 飛簷走壁）
        │
        │  轉化規則：
        │   1. 顏色 → 純黑底色
        │   2. 輪廓 → 單色光條
        │   3. 質感 → 發光材質
        │   4. 動作 → 加流光殘影
        │
        ▼
GlowFrame 戰甲設計
  ├─ 戰甲（袍 / 鎧版本的賽博化）
  ├─ 流光武器（劍刀槍的光條版）
  ├─ 流光配件（玉佩 → 發光徽章）
  └─ 武術動作（武俠招式 + 流光殘影）
```

### 案例對應

| 武俠原型 | GlowFrame 版本 |
|---|---|
| 白衣劍俠 | 白光條戰甲、流光長劍 |
| 紅衣女俠 | 紅光條戰甲、流光雙刀 |
| 黑衣刺客 | 紫光條戰甲、流光匕首（**初號機可能就是這型**）|
| 道士 | 金光條戰甲、流光拂塵 |
| 武僧 | 橙光條戰甲、流光禪杖 / 拳套 |
| 弓箭手 | 綠光條戰甲、流光長弓 |
| 樂師 | 藍光條戰甲、流光樂器 |
| 暗器專家 | 銀光條戰甲、流光暗器（針 / 刀 / 鉤）|

→ **8-12 種武俠類型 × 流光化 = 8-12 個戰甲款式、邊際成本極低**。

## 工具流（colombo 既有經驗）

| 工具 | 用途 | 限制 |
|---|---|---|
| **MagicaVoxel** | voxel 模型本體 | 純靜態、無動畫 |
| **BlockBench** | 動作 / 動畫 | **動作不能重複利用**、每動作要從頭做 |
| **Blender**（需引入）| 流光特效 + 動畫複用 | 學習曲線、但解 BB 限制 |

未來真要動工：
- voxel 模型還是 MagicaVoxel（風格匹配）
- 動畫 / 流光特效改 Blender
- 統一導 Godot / Unity 引擎

## 配色 IP 系統

未來戰甲命名建議：

```
[流螢戰甲] [流派 / 顏色] [編號]
例：
  流螢戰甲 紫影系列 初號機 ← 已有 voxel
  流螢戰甲 朱劍系列 二號機
  流螢戰甲 青弓系列 一號機
  流螢戰甲 金禪系列 一號機
  ...
```

→ 每個系列 = 一個武俠流派 + 一個專屬顏色 = 一條變現支線（皮膚 / 戰甲套裝 / 角色 IP）。

## 美術風格的 scalability

```
傳統高解析 3D 模型：
  每個新角色 = 重新建模 + 貼圖 + rigging + 動畫
  邊際成本：50-200 工時 / 角色

GlowFrame voxel 流光：
  每個新角色 = 新光條顏色 + 些許剪影差異
  邊際成本：5-20 工時 / 角色

→ scalability 10x。
→ 適合 indie 開發 / 小團隊 / colombo 一人做。
```

→ **這個美術系統本身就是 IP 商業可行性的關鍵**——因為**便宜可拓展**。

## 跟造物森林美術系統的對比

| 維度 | 造物森林 | GlowFrame |
|---|---|---|
| 風格 | 萌物 + 現實物件 | 黑底光條 + 武俠動作 |
| 角色製作成本 | 中等（每隻獨特造型） | 低（光條顏色換、剪影微調）|
| 視覺辨識度 | 物種 + 物件 = 兩重 | 單色光條 = 一重 |
| 擴張邏輯 | 加新物種 + 新垃圾 | 加新流派 + 新顏色 |
| 玩家辨識 | 直觀（看就知道是樹蛙）| 需要學習（紫光條 = 暗影系）|

→ 兩 IP 美術系統互補、可共用 voxel pipeline、不互相排斥。

## 跟 LL 視覺品牌的關係

- LL 主視覺 = 月衍卿風格、暫定
- 造物森林 = LL 接地產品線
- GlowFrame = LL 離地 / 後人類產品線
- 三者都是 voxel 路線、但風格分明
- 未來 LL 對外形象 = **多風格 IP 母艦、不是單一風格**
