# 大富翁（Richman）設計綱要

> 狀態：構想期（2026-07-11 立檔）。需求來源：女友喜歡大富翁——第一個玩家就在家裡。
> 資料夾暫名 richman（「大富翁」通行英譯；Monopoly 為 Hasbro 註冊商標、上架避用）。

## 一句話定位

OpenMoji ＋ grid 就能出畫面的全家同樂棋盤遊戲；**state 複雜、action 極小**，天生適合
RL 訓練境（RR 線）與 LLM 對手（souls-hall 線）。素材走 MojiSmith（MS）。

## 畫面＝OpenMoji + Grid

- 棋盤＝一圈 grid 路徑；地格/建物(🏠🏨)/棋子/機會命運(❓🎲)/車站公司(🚂💡) 全 emoji
- 素材需求極淺：不需 rig、不需花紋——**MS 素材線最早能養活的遊戲**（對照：牧場物語要等 rig）

## Action 空間分析（核心洞察：大部分回合沒有決策）

完整 action 只有七類；走格/抽卡/付租全是強制結算、不是 action：

| # | Action | 參數 | 時機 |
|---|---|---|---|
| 1 | 擲骰 | — | 每回合（強制、零決策） |
| 2 | 買/不買 | 二元 | 落在無主地 |
| 3 | 蓋房 | 選地 | 同色組成套後 |
| 4 | 賣房/抵押 | 選地 | 缺錢 |
| 5 | 贖回 | 選地 | 有錢 |
| 6 | 出獄 | 付錢/用卡/擲骰 | 獄中 |
| 7 | 交易 | 對象＋內容 | 唯一的大空間 |

**三段縮法**：
1. 砍交易＋拍賣（台式數位版慣例）→ 剩 買/不買＋資產管理＋出獄三選一
2. 資產管理自動化（缺錢自動抵押最便宜、閒錢自動蓋最貴）→ **只剩「買/不買」一個二元決策**
3. action masking：任一時刻合法動作 2～5 個 → UI 只亮當下能按的鈕

## 生態接點（一魚多吃）

- **RR / RL**：state 大 × action 小＝教科書訓練境；Q-table 可玩；「靈寵學理財」
- **souls-hall LLM 對手**：造句召喚師的逐步決策協定直接複用——每個決策點問一次「買不買」
- **勁園教材線**：理財教育角度（現金流/資產/風險），可掛學習目標＋評分
- **多人**：全家同樂＝最初需求；未來可接 SS 房間機制

## 策略深度設計（2026-07-11 拍題：不做好看的擲骰子模擬器）

**核心判斷**：大富翁淪為骰子模擬器，是因為數位版把真正的策略核心——**交易談判**——砍了；
砍它不是設計判斷、是腳本 AI 不會談判的技術限制。**LLM 恰好解鎖的就是這塊。**

### 三層樓（按段位開鎖、不推翻極簡 MVP）

1. **機率與期望值**：監獄＝全盤熱點、出獄後 6–8 步（橘/紅組）被踩率最高、各色組 ROI 可算
   → 盤面疊**熱區圖**、地產卡標 ROI。期望值教育是「玩出來的」。
2. **現金流與風險**：留現金 vs 蓋房的流動性兩難
   → **財商教練**（造句裁判模式直接複用、決策後點評不擋操作：「現金剩200還買地＝流動性風險」）
   → **局後復盤**：淨資產曲線、租金收入、轉折點——即范總要的學習歷程、一局一份財商報告。
3. **談判桌（LLM 才開得了的門）**：自然語言交易——「我用綠色那塊換你的火車站、再貼500」
   → LLM 在角色裡評估/還價/小心機；市面數位大富翁均無此物。
   技術＝造句召喚師「句子→動作」協定＋offer 解析（已會）。談判本身＝財商（估值/讓步/雙贏vs零和）。

### 四獸財商性格（對手本身是教材；souls-hall 各給性格銘文）

| 棋子 | 性格 | 教的課 |
|---|---|---|
| 🐻 熊 | 囤現金不敢投資 | 太保守＝被通膨磨死 |
| 🦊 狐狸 | 激進槓桿梭哈 | 爆發強、踩旅館連環爆——槓桿雙面刃 |
| 🐶 狗 | 穩健、成套才蓋 | 教科書玩法、活最久 |
| 🐱 貓 | 談判型、以換代買 | 交易創造價值 |

輸給狐狸和輸給熊是兩堂不同的課。台詞經 souls-hall 性格銘文自然流露其邏輯。

### 段位開鎖（同造句召喚師「教學→訓練→對戰」樓梯）

家庭場（自動管理、擲骰＋買/不買）→ 進階場（手動蓋房/抵押＋熱區圖＋財商教練）→ 談判場（交易解鎖＋四獸全開）。

## 去監獄化：商圈機制（2026-07-11 colombo 拍板）

**洞察**：監獄機制的本質＝人流製造機（全盤最熱格、橘色組因蹲在出獄後6-8步而最值錢）。
原版把「人流＝地價」藏在監獄皮下；換成商圈直接把課翻上檯面，負面格清零（同環境保衛戰「擊退不擊殺」的品味）。

| 原版監獄 | 商圈版 | 保留本質 |
|---|---|---|
| 入獄格/入獄卡→傳送 | 週年慶格/血拼日卡→**強制進百貨消費** | 人流磁鐵 |
| 蹲牢1-3回合 | 逛百貨出不來（排隊結帳） | 節奏懲罰 |
| 付$50出獄 | 直接結帳走人 | 用錢買時間 |
| 擲雙數出獄 | 擲雙數＝輪到你結帳 | 賭運氣 |
| 出獄卡 | VIP 快速通道卡 | 稀缺資源卡 |
| 監獄旁橘組最熱 | **百貨商圈周邊地價/租金加成（明示係數）** | 人流→地價、變看得見的課 |

**便利店**（另一條線）：**經過**時擲骰、機率性進去花小錢 → 局後復盤統計「本局便利店共花$450」＝**拿鐵因子**課（小額消費的累積殺傷力）。

**對稱延伸（待議）**：
- 便利店可被買下、盤上4家接手原版車站×4位置（連鎖：家數越多抽成越高）→ 中心思想閉環：**同一家店、消費側掏你錢、資產側收別人錢**（財商第一課）
- 百貨也可有主人（原版監獄無擁有權、此為加法）：強制消費進老闆口袋＝「買下人流本身」；放談判場再開鎖

**牧場物語判斷（同日）**：LLM 加成低（只剩村民聊天風味）、教學內涵薄 → 降序、留點子庫。

## MVP 邊界（防外溢）

- 先做：單機、4 玩家（1 人＋3 AI）、無交易、無拍賣、資產管理自動化 → 玩家只按「擲骰」和「買/不買」（＝家庭場）
- 後補：手動蓋房、熱區圖、財商教練、談判場（LLM 交易）、聯機
- 不做：房屋短缺規則、複式拍賣（原版深水區、全年齡不需要）
